# Nexus HSB 管理界面产品需求文档

## 1. 文档目标

本文档用于定义 Nexus HSB 管理界面的整体功能结构，明确：

- 界面需要围绕哪些核心对象进行维护
- 各对象支持哪些增删改查与运行处置动作
- 主要业务流程如何在界面中落地
- 一级、二级和功能菜单如何组织
- 关键页面需要展示哪些字段、提供哪些按钮动作、配置哪些权限点

本文档面向产品、设计、前端、后端和实施团队，作为管理后台建设的统一基线。

## 2. 产品定位

Nexus HSB 是面向医院集成场景的服务总线平台。管理界面需要同时支持四类能力：

- 配置设计：端点、路由、转换器、工作流、接入通道
- 运行监控：消息、端点、路由、工作流、通道、熔断器、指标
- 治理处置：消息追踪、死信重放、审计查询、异常排障
- 安全控制：SSO、权限、密钥、证书、敏感操作审计

管理界面不只是“配置后台”，还应承担“运维驾驶舱”和“故障处置平台”的角色。

## 3. 目标用户与角色

### 3.1 目标用户

- 集成实施工程师：负责新系统接入、端点和路由配置
- 运维工程师：负责运行监控、异常处理、死信重放
- 平台管理员：负责全局配置、权限、发布、回滚
- 安全管理员：负责 SSO、证书、密钥、审计
- 业务管理员：负责查看消息流转、工作流执行结果、审计报表

### 3.2 建议角色

- 超级管理员
- 平台管理员
- 集成管理员
- 运维管理员
- 安全管理员

## 5. 核心业务流程定义

### 5.1 新系统接入流程

1. 新建端点
2. 配置连接参数、认证参数、安全参数
3. 执行连通性测试
4. 新建路由
5. 配置源匹配、消息类型、目标端点、投递策略
6. 绑定转换发布包
7. 发布配置
8. 发送测试报文
9. 查看消息轨迹与审计日志
10. 正式启用

### 5.2 消息异常处置流程

1. 从首页告警或消息监控进入异常对象
2. 查看消息详情与轨迹
3. 判断失败原因是端点、路由、转换还是下游错误
4. 修复相关配置
5. 单条重处理或死信重放
6. 验证结果
7. 记录处置结论

### 5.3 路由变更流程

1. 查询当前路由
2. 查看关联端点和命中统计
3. 修改匹配条件、目标、优先级或投递模式
4. 执行仿真测试
5. 保存草稿
6. 审批发布
7. 观察运行指标
8. 必要时回滚

### 5.4 端点变更流程

1. 查询端点
2. 修改地址、超时、认证、证书等参数
3. 保存新版本
4. 健康检查
5. 影响分析
6. 发布生效
7. 观察成功率和错误率
8. 必要时回滚版本

### 5.5 工作流编排流程

1. 新建工作流
2. 设计步骤与跳转关系
3. 配置补偿步骤和重试策略
4. 执行测试
5. 发布启用
6. 查看实例运行情况
7. 对失败实例执行恢复或补偿

### 5.6 配置发布流程

1. 修改草稿
2. 进行配置校验
3. 查看差异对比
4. 提交审批
5. 发布配置
6. 重载生效
7. 观察运行状态
8. 必要时回滚
